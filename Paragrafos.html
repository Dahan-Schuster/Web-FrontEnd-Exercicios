Boas práticas de programação - Parte 2

Dicas para construção de código limpo e organizado

Crie sistemas que sejam mantíveis em primeiro lugar.

Torne seu código legível da mesma forma como o torna executável. Lembre-se: alguém irá ler e atualizar seu sistema algum dia, seja este alguém você ou não, então torne-o de fácil compreensão.

Refatore! Refatore! Refatore!

Seja honesto com seu código. Refatoração possui diversos benefícios, como:

  • Tornar o código organizado;
  • Menor probabilidade de bugs (teste e comprove: muitas vezes um bug ocorre simplesmente por desorganização do código)
  • Facilidade para encontrar bugs (além de possuir poucos bugs, um código organizado torna fácil a procura pelos que ainda restam);
  • Aumentar o desempenho.

Escreva seus códigos como escreveria um livro.

Uma boa forma de tornar seu código organizado é mapeá-lo como um livro, nomeando cada método como nomearia um capítulo, cada variável como um personagem.
Por exemplo, não é necessário ver o código inteiro para saber o que este sistema faz apenas lendo seus métodos principais:

        
        public static void main(String[] args) {
            abrirArquivo();
            recuperarDadosDoArquivo();
            preencherListaComDados();
            receberDadosDoUsuario();
            salvarDadosNoArquivo();
            fecharArquivo();
        }

Faça um teste antes de ler a linha abaixo: quanto tempo você demora para entender o que faz este programa?

Resposta: um sistema que manipula um arquivo .txt, lê nomes digitados pelo usuário, armazena em um ArrayList e salva no arquivo.
A sensação que dá é nem seria necessário explicar, não é? Isto porque o programa já é autoexplicativo.

Use nomes pronunciáveis.

Humanos são bons em decifrar palavras pronunciáveis, então nomeie seu código com palavras que nós possamos entender. Por exemplo, use "valorMaximo" ao invés de "vlrMx".
